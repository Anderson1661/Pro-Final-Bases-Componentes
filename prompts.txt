# Prompts para Gemini CLI - Mantenimiento del Proyecto "Aventureros S.A."

Este archivo contiene una serie de prompts listos para ser usados con Gemini CLI para facilitar el mantenimiento y actualización de este proyecto.

---

### 1. Prompt para Actualizar el README.md y Funcionalidades

**Objetivo:** Si se han añadido nuevas funcionalidades, archivos o se ha cambiado la estructura, este prompt instruirá a Gemini para que re-analice el proyecto y actualice la documentación.

**Prompt:**
"Hola Gemini. Necesito que actualices la documentación del proyecto. Por favor, analiza toda la estructura de directorios, el archivo `Base/base.sql`, los archivos de la aplicación en `Transportadora/` y los archivos de la API en `conexiones_bd/`. Luego, actualiza el archivo `Funcionalidades.txt` con cualquier nueva funcionalidad que detectes y, finalmente, regenera el `README.md` para que refleje el estado actual y completo del proyecto, incluyendo la estructura de directorios, la guía de la API y las funcionalidades."

---

### 2. Prompt para Regenerar/Actualizar los Endpoints de la API por Cambios en la BD

**Objetivo:** Si se modifica el archivo `Base/base.sql` (añadiendo tablas, eliminando vistas, cambiando columnas), este prompt le pedirá a Gemini que regenere todos los archivos PHP de la API para que coincidan con el nuevo esquema.

**Prompt:**
"Hola Gemini. He modificado el esquema de la base de datos en `Base/base.sql`. Necesito que vuelvas a analizar este archivo para identificar todas las tablas y vistas. Luego, crea la estructura de carpetas y archivos CRUD (create, read, update, delete) para cada nueva tabla, y el archivo `read.php` para cada nueva vista dentro de `conexiones_bd/`. Asegúrate de que los nuevos scripts PHP coincidan exactamente con las nuevas columnas y nombres."

---

### 3. Prompt para Configurar la Conexión a la Base de Datos en la Nube

**Objetivo:** Cuando la base de datos se mueva de un entorno local a un servidor en la nube (como Render, Neon, etc.), este prompt guiará en la actualización del archivo de conexión.

**Prompt:**
"Hola Gemini. He migrado la base de datos a un servidor en la nube. Necesito que modifiques el archivo `conexiones_bd/conexion.php`. Por favor, cambia las variables `$host`, `$port`, `$dbname`, `$user` y `$password` con los siguientes valores: [PEGAR AQUÍ LOS NUEVOS VALORES DE CONEXIÓN]. Asegúrate de no alterar el resto del código del archivo."

---

### 4. Prompt para Añadir una Nueva Funcionalidad (Ejemplo: Calificaciones)

**Objetivo:** Guía para añadir una nueva característica de punta a punta (BD -> Backend -> Documentación).

**Prompt:**
"Hola Gemini, quiero añadir un sistema de calificaciones de servicios. El plan es:
1.  **Base de Datos:** Modifica `Base/base.sql` para añadir una nueva tabla `calificacion_servicio` que tenga `id_calificacion` (SERIAL PK), `id_ruta` (FK a ruta), `puntuacion` (int, 1-5), y `comentario` (TEXT).
2.  **API Backend:** Crea la carpeta y los archivos CRUD completos para la nueva tabla `calificacion_servicio` dentro de `conexiones_bd/`.
3.  **Documentación:** Actualiza `Funcionalidades.txt` y `README.md` para incluir esta nueva característica en el módulo de Cliente (para dejar una calificación) y en el de Administrador (para ver las calificaciones)."

---

### 5. Prompt para Explicar una Parte del Código

**Objetivo:** Para obtener una explicación detallada de cómo funciona un archivo o script específico.

**Prompt:**
"Hola Gemini. No entiendo bien cómo funciona una parte del proyecto. Por favor, lee el archivo [PEGAR RUTA ABSOLUTA DEL ARCHIVO, ej: /Users/macbook/Documents/GitHub/Pro-Final-Bases-Componentes/conexiones_bd/ruta/create.php] y explícame en detalle qué hace, cómo procesa los datos de entrada y cómo interactúa con la base de datos."
